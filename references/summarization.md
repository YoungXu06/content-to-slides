# 5-Question Summarization Framework

读完原文/听完原片后，在心里用**5 问骨架法**回答下面的问题。**答案决定 slides 骨架**。写不出来就是没读透，回去重读；绝不要在没答案的情况下硬写 slides。

## 5 问骨架法

回答这 5 个问题（内部思考，不必输出给用户）：

| # | 问题 | 用于 |
|---|------|------|
| 1 | **一句话核心** ≤ 25 字：这篇/这段内容到底想让观众记住哪一件事？ | 钩子页 / 收束页 |
| 2 | **痛点钩子**：观众原本困在什么地方？为什么会点进来？ | 钩子页（Slide 1） |
| 3 | **3-5 个关键论点**：支撑核心的骨头（不是所有段落都是骨头） | 论点页（Slide 3-6） |
| 4 | **最值钱的 1 个反直觉点**：原文里最容易被忽略但最有价值的 insight | 反直觉页（Slide 7） |
| 5 | **能改变什么**：看完观众可以做什么 / 改变什么看法？ | Takeaway 页（Slide 8） |

---

## 每问的判断标准

### Q1: 一句话核心

- 必须 ≤ 25 字，长了说明没抓到骨
- 不能是原文标题的复读。标题是钓鱼，核心是答案
- 好例：「AI 协作的瓶颈不是模型，是上下文管理」
- 坏例：「本文探讨了 AI 编程中协作模式的最新进展」（这是摘要开头，不是核心）

### Q2: 痛点钩子

- 要能让刷到视频的人在 3 秒内觉得"这跟我有关"
- 可以是一个尴尬经历、一个反复踩的坑、一个常见误解
- 好例：「你 AI 用了一年，为啥别人效率翻倍你还在删删改改？」
- 坏例：「AI 是当今最重要的技术趋势之一」（和观众零关系）

### Q3: 3-5 个关键论点

- **必须是论点，不是话题**。「上下文管理」是话题；「把任务拆小到 AI 一次能看完是根本」才是论点
- 每个论点独立成页，互相不重叠
- 如果原文有 10 个要点，压到 3-5 个，**砍掉重复和边角**
- 数量不够 3 个时：这内容太薄做不出 5 页，退回 Phase 2 问用户要不要补

### Q4: 反直觉点

- "最值钱" ≠ "最核心"。最核心的是 Q1；反直觉的是 Q4
- 它往往是原文中一句轻描淡写但让你停顿的话
- 找不到反直觉点 → 把原文最容易被误解的地方拎出来当这一页
- 它是整个讲解里最容易被收藏、被转发的一页

### Q5: 能改变什么

- **具体行动 > 抽象态度**。"要多尝试" 是废话；"今晚把你最常用的 AI prompt 加一行具体约束看效果" 才是行动
- 抽象观点也行，但要够锐（"以后别再信 XXX"）
- 避免鸡汤结尾

## 把答案映射到 slides

```
Q1 (一句话核心)     → Slide 1 钩子的靶心  +  Slide N-1 收束回扣
Q2 (痛点钩子)       → Slide 1 开场
Q3 (3-5 个论点)     → Slide 3, 4, 5 (, 6)
Q4 (反直觉点)       → 倒数第 2-3 页，视觉上放大
Q5 (能改变什么)     → 倒数第 2 页
```

"是什么 / 定义页" (Slide 2) 是可选的 —— 只有当核心概念需要先解释术语时才加。

## 常见提炼错误

| 错误 | 症状 | 修正 |
|------|------|------|
| **把摘要当核心** | Q1 写成"本文介绍了..." | 再读一遍，用 ≤ 25 字的断言句改写 |
| **论点罗列成分类** | Q3 变成"原因 / 方法 / 影响" | 把每一类里最锋利的那个具体论点抓出来 |
| **没找反直觉点** | Q4 写了和 Q3 雷同的东西 | 主动找原文里"等等，这句话听起来不太对但想想挺对"的地方 |
| **takeaway 是鸡汤** | "保持学习、保持好奇" | 换成具体行动或具体认知转变 |
| **原文太水硬拔骨** | 真挤不出 3 个论点 | 停下告诉用户"这篇内容撑不起 5 页讲解，要不补点材料？" |

## 真实示例

**输入**：一篇讲"为什么远程工作并没有提升多数公司的效率"的文章（假设 2000 字）。

```
Q1 一句话核心: 远程没失败,是"把线下流程搬到线上"失败了
Q2 痛点钩子:  你公司远程两年了,会议更多、产出更少,为啥?
Q3 3 个论点:
  ① 信任成本被低估:管理者不敢放手,反而加码监控
  ② 异步写作文化没建立:还在用同步会议的肌肉记忆工作
  ③ 办公室的"走廊对齐"消失了,但没人补上它
Q4 反直觉点:  问题不在员工偷懒,在于"中层管理者的工作方式"
              —— 他们赖以生存的协调机制被直接拆了
Q5 能改变什么: 今晚看一眼上周的会议列表,把能变成一封信的全砍掉
```

对应的 slides 结构：

```
P1 [钩子]    "你公司远程两年,为啥更累了?"
P2 [是什么]   远程工作 vs 远程办公
P3 [论点]    信任成本:监控不等于产出
P4 [论点]    异步不是"用微信开会"
P5 [论点]    走廊对齐消失了,但没人补
P6 [反直觉]   真正的危机在中层
P7 [行动]    这周砍一场本可以不开的会
P8 [收束]    远程没失败,是你搬错了办公室
```

这份骨架进 Phase 3 后，每页标题 + 每页一句话核心就成型了。
